# Rig Signal Lab

Application ID: `HTH-RS-001`

Rig Signal separates five questions that must never be collapsed into one score:

1. What is the device or component?
2. What maturity state is it in?
3. What capability is actually documented or validated?
4. What data can be imported safely?
5. What, if anything, is suitable for a fishing recommendation?

## Current implementation

- versioned sensor-agnostic `device-event.v1.json` contract;
- explicit units, quality, calibration, uncertainty, privacy, retention, source, and location-free output;
- separate maturity and capability-evidence fields;
- fail-closed unknown privacy classes;
- manual logging remains the complete baseline.

## Autonomous progression

Contract additions, fixtures, adapters, tests, documentation, and preview interfaces proceed automatically under the active AI autonomy standard when reversible and passing applicable checks.

A narrow exception review is required only for:

- credentials or protected account access;
- new handling of precise location or sensitive personal data;
- manufacturer or performance claims unsupported by evidence;
- binding partnership, payment, or legal commitments;
- irreversible production actions or unresolved material safety risk.

Manufacturer documentation establishes a documented claim. It does not prove field accuracy, bite detection, species identification, fight classification, or improved catch outcomes.
