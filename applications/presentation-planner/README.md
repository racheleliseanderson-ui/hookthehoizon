# Presentation Planner

Application ID: `HTH-PP-001`  
Adapter key: `horizon.presentation-planner`  
Status: draft MVP scaffold (`0.1.0-planning`)

Canonical authority: Google Drive — **Hook the Horizon — Three Intelligence Hubs and Web Applications — Build Extension Specification — v1.0 — 2026-07-16**.

## Reader job

Use declared target, water and field conditions, structure, depth, access mode, trip goal, owned inventory, and equipment constraints to produce a small set of explainable presentation experiments.

## Boundaries

- Inventory-first; affiliate value and popularity do not change a result.
- No exact location is required, retained, logged, or exposed.
- No catch, activity, safety, or legal-use guarantee.
- Equipment-fit questions route to `HTH-SC-001 System Compatibility`.
- Regulations, closures, access, conservation, and sensitive-location issues route through Honey Hole Intelligence controls.
- The application must show factors and limits rather than a theatrical one-number score.

## First result contract

- up to three plans using owned inventory
- lure, bait, fly, or terminal class
- size and color family
- depth-control approach
- retrieve, drift, cadence, pause, speed, or angle plan
- reason each variable matches the declared conditions
- failure signals and first adjustment
- minimal trip pack with redundancy logic
- no-purchase or use-what-you-own alternative
- evidence, confidence, and unknown states
- private post-trip feedback prompt

## Planned implementation

1. Define controlled presentation, action, depth, clarity, structure, forage, and outcome vocabularies.
2. Add a versioned inventory, candidate, input, and result contract.
3. Add a deterministic evaluator that hard-excludes equipment mismatches and ranks supplied evidence-backed candidates.
4. Correct for popularity and reporting bias; negative outcomes remain first-class data.
5. Add fixtures for complete, incomplete, conflicting, stale, mismatch, and exact-location inputs.
6. Add privacy, ranking, no-purchase, and accessibility tests.
7. Build an accessible mobile-first preview.

The application consumes shared evidence, confidence, provenance, saved-state, print/share, analytics, and privacy behavior from `@editorial-ecosystem/intelligence-core` where useful.
