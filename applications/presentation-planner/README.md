# Presentation Planner + Horizon Smart Mode

Application IDs:

- `HTH-PP-001` — Presentation Planner
- `HTH-SM-001` — Horizon Smart Mode personalization layer

Status: autonomous preview implementation under the active AI Autonomy and Exception Governance Standard.

## Reader job

Use broad declared conditions, owned inventory, equipment-compatibility state, reviewed source records, and bounded personal outcome history to produce:

1. a best-supported presentation;
2. a deliberately different contrast experiment;
3. a fallback plan;
4. one major variable to change first;
5. a no-purchase route;
6. a private mastery summary.

## Implemented capabilities

- deterministic, factor-explained personalization;
- local inventory and outcome history;
- reporting-volume bias cap;
- hard equipment-mismatch exclusion;
- public-safe share-card export;
- local progress and mastery summary;
- offline-capable PWA preview;
- local knowledge-graph cross-links;
- no account, external API, LLM, precise location, telemetry, or purchase required.

## Privacy boundary

Exact coordinates, map links, precise waterbody identifiers, private access details, and home-waterbody IDs are rejected. Profile, inventory, recommendation history, and outcomes stay on the local device in this preview.

## Run

Serve the repository root from a local HTTP server and open:

`/applications/presentation-planner/preview/`

The service worker only caches this bounded preview surface. It does not intercept the WordPress site.

## Autonomous release profile

Routine code, tests, preview builds, and source merges proceed automatically when the affected checks pass. Exhaustive universal QA and generic human approval are not required. Tier 3 exceptions remain narrow: sensitive location, protected personal data, credentials, binding policy, unsupported safety/regulation/hardware claims, irreversible actions, or materially unresolved risk.
