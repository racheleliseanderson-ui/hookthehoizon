# Governing Information Principle

Status: active implementation contract  
Effective: 2026-07-16

This repository follows the Editorial Ecosystem AI Autonomy and Exception Governance Standard.

When AI can safely obtain, verify, or automate the collection of required information with a reasonable level of confidence, it must do so. The result must be consolidated into the designated authoritative source containing the best currently known information.

Every material conclusion or implementation decision must preserve proportionate evidence and clearly identify, where relevant:

- source and verification date;
- verification method and freshness;
- confidence level: High, Moderate, Low, or Unknown;
- assumptions and limitations;
- unresolved uncertainty or conflict;
- applicable tests, acceptance evidence, and rollback path;
- the controlling Drive authority and downstream implementation references.

Do not create an unmarked competing source of truth. Repository files, issues, pull requests, code, tests, release records, and WordPress implementations must reference or operationalize the active authority. When sources conflict, reconcile them using authority, recency, evidence quality, and scope; classify superseded or reference-only material explicitly.

Proceed automatically for governed, measurable, reversible work within established limits. Escalate only the smallest genuine legal, privacy, security, financial, credential, sensitive-data, destructive, irreversible, qualified-professional, or materially harmful exception.

Acceptance requires one active authoritative record, traceable evidence, visible confidence and uncertainty, aligned implementation, and passing applicable QA.

**Final rule:** obtain what can be safely obtained; verify what can be verified; automate what can be reliably repeated; consolidate the result into one authoritative source; and present the best currently known information with evidence, honest confidence, explicit limitations, and visible uncertainty.
